Medidas Implementadas

1. Rate Limiting (slowapi)
Límite de 30 requests/minuto por IP en endpoints de proceso y exportación. Previene abuso del OCR y DoS locales.
2. Sanitización de Paths
Todos los paths de archivos se resuelven con Path.resolve() y se verifica que estén dentro del directorio de uploads. Se rechazan paths con .. o enlaces simbólicos.
3. Sanitización de Nombres
Los nombres de archivo generados por /api/rename se limpian de caracteres peligrosos: ;&|$`(){}[]!#. Se usa re.sub para eliminar cualquier carácter no alfanumérico excepto espacios, guiones y puntos.
4. Seguridad en Headers HTTP
Se agregan headers de seguridad: Content-Security-Policy, X-Content-Type-Options: nosniff, X-Frame-Options: DENY, Strict-Transport-Security.
5. Validación de Archivos
Solo se aceptan archivos con extensión .pdf. Se verifica el magic number del PDF (%PDF) antes de procesar.
6. Sin Dependencias Innecesarias
No se usan librerías que introduzcan vectores de ataque (no hay XML, no hay pickle, no hay YAML, no hay exec/eval).

Lo que NO es Aplicable (por arquitectura)

  • Network Hacking — La app solo escucha en 127.0.0.1, no accesible desde la red.
  • Wireless Hacking — No hay componente WiFi.
  • Cloud Attacks — Sin conexión a cloud, sin metadata endpoints.
  • Mobile Hacking — No es una app mobile.
  • Hardware/Firmware — Software puro, sin firmware.
  • Reverse Engineering — El .exe es de PyInstaller, el código es Python. Quien tenga acceso físico a la PC puede leerlo, pero eso es inherente al modelo de distribución.
  • CMS Exploitation — No hay CMS.
  • Supply Chain — No hay dependencias externas en runtime.
  • Ingeniería Social — No hay interacción con usuarios remotos.
  • C2 / Persistence — No hay comunicación externa.

Recomendaciones Adicionales

  1. Firmar el .exe con certificado digital para evitar advertencias de Windows Defender.
  2. No almacenar datos sensibles en texto plano si se agrega persistencia en el futuro.
  3. Ejecutar con permisos mínimos — no requiere admin para operar.
  4. Revisar dependencias periódicamente con pip audit o safety.
